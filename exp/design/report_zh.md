# 设计批评：为什么单次切分回测会失败，以及该如何正确评估

## 1. 核心问题不是制度变迁——而是评估协议本身

消融研究中的 H2（反向时间切分）表明，在 2020–2022 年高波动制度上学到的参数，能够*反向*泛化到 2014–2019 年的趋势性制度。这种不对称性通常被解读为"市场变了"。

但更精确的解读是：**单次训练/测试切分是错误的工作。**

金融时间序列在构造上就是非平稳的。任何一个固定的切分日期都是武断的。一个恰好与训练制度对齐的策略会看起来很出色；一个不对齐的策略会看起来很失败。主导 headline 数字的是切分的运气，而非算法的优点。

这不是推测性的主张。Lopez de Prado (2018) 已经证明，标准 k 折交叉验证在金融时间序列上是**无效的**，因为它通过重叠的标签将未来信息泄露到训练集中。正确的协议是**前向验证**（滚动或扩展窗口），即每一步都只用该时刻之前可得的数据重新训练模型。

本目录中的前向验证实验（`walk_forward.py`）实现了滚动 3 年训练 / 1 年测试的协议。结果是清醒的：

| 窗口 | 买入并持有 | PSO | GP | PSO > BH? | GP > BH? |
|------|-----------|-----|----|-----------|-----------|
| 2017–2018 | $1,711 | $1,000 | $590 | 否 | 否 |
| 2020–2021 | $4,106 | $1,000 | $4,106 | 否 | 平 |

在两个窗口中，PSO 从未击败买入并持有。GP 在一个窗口中与之持平，在另一个窗口中崩溃。单次切分实验中"PSO train=$53k"的 headline 因此**不是一个稳健的发现有**——它仅仅是制度对齐的伪影。

## 2. 买入并持有是错误基准

买入并持有在主动管理的意义上不是一个"策略"；它是一种**被动的 beta 暴露**。它捕捉的是资产的长期漂移，由宏观采用、货币政策、减半周期和社会情绪驱动——这些因素与 PSO 和 GP 正在优化的微观结构信号（交叉检测、动量）毫无关系。

因此，将主动交易算法与买入并持有进行比较是一个**范畴错误**。它混淆了两个不同的问题：
1. 你能否比抛硬币更好地择时？（技能，alpha）
2. 你是否骑上了长期牛市？（暴露，beta）

主动策略的适当基准是**其相对于具有相同风险暴露的被动基准的超额收益**，以跟踪误差标准化。这就是**信息比率（IR）** [1]：

```
IR = (Return_strategy − Return_benchmark) / TrackingError
```

等价地，可以使用**詹森 Alpha** [2]，它将策略收益对基准收益进行回归，并检验截距是否统计显著：

```
R_strategy = α + β · R_benchmark + ε
```

如果 α ≤ 0，则策略没有增加被动暴露之外的价值。在我们的前向验证结果中，相对于买入并持有，PSO 和 GP 均表现出负的或接近零的 alpha。

## 3. "区间内的胜率"是一个有效但未被充分利用的指标

用户建议通过策略在**区间内的胜率**来评估算法——即在策略跑赢基准的时间窗口（天、周、月）中所占的比例。

这在概念上是合理的，并且在文献中有先例。策略的**击球率**或**命中率**衡量正确方向判断的频率 [3]。当与**盈亏比**（平均盈利 / 平均亏损）结合时，可以得出**凯利准则**最优比例和策略的**期望值**：

```
Expectancy = (WinRate × AvgWin) − (LossRate × AvgLoss)
```

一个高胜率但负期望值的策略是危险的：大多数时候感觉很好，但偶尔的大亏损会抹掉所有收益。相反，一个低胜率但正期望值的策略（例如趋势跟踪）如果仓位管理严格，也可以是盈利的。

在我们的语境中，"区间胜率"应定义为：
- **逐笔胜率**：盈利交易占所有交易的比例。
- **逐期胜率**：策略收益 > 基准收益的前向验证窗口占所有窗口的比例。

前向验证实验报告的是第二种定义。PSO 的得分为 0%，GP 的得分为 0%（如果平局算赢则是 50%）。这比单次切分的 headline 更诚实。

## 4. 综合：一个正确的管道应该长什么样？

| 组件 | 当前（错误） | 正确 |
|------|------------|------|
| 数据切分 | 2020 年的单一静态切分 | 前向验证滚动/扩展窗口 |
| 基准 | 买入并持有（被动 beta） | 风险调整后的超额收益（IR、Alpha） |
| 适应度 | 最终现金 | 夏普、卡玛或索提诺比率 |
| 评估 | 一个测试集 | 多个窗口 + 区间胜率 |
| 制度 | 被忽略 | 优化前的显式检测（马尔可夫转换） |
