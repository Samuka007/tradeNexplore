# 设计批评：为什么单次切分回测会失败

## 4. 结论：关于"外力因素"与技术分析的反思


用户指出："外部冲击只是事后诸葛亮的说法，我想要的是在外力因素基础上的技术分析。"

这个批评是准确的。我们不应该用"BTC 有外部冲击"来解释策略失败——**所有市场都有外部冲击**，技术分析本身就是要在不知道具体"为什么"的情况下，从价格序列中提取模式。

真正的问题不是"有没有外部冲击"，而是：**WMA 交叉这种低阶线性信号，能否从 BTC 的价格动态中提取出可预测的模式？**

答案是否定的，原因有三：

### 4.1 制度非平稳性
BTC 的收益分布在不同时间段有显著差异。但这不是"外部冲击"的问题——而是**任何固定参数的策略都无法适应非平稳数据**。即使不知道具体什么时候会断裂，也知道它*会*断裂。鲁棒优化试图通过跨窗口平均来缓解这个问题，但效果有限。

### 4.2 凸性不对称
买入并持有具有**正凸性**：牛市时跟上涨幅，熊市时最多归零（本金保全）。主动交易策略（基于交叉信号）具有**负凸性**：牛市时频繁交易错过涨幅，熊市时频繁止损放大亏损。这种不对称性与具体的外部冲击无关，是策略结构本身的数学性质。

### 4.3 信号与噪声的比率
BTC 日收益率的信噪比极低。WMA 交叉等低阶信号无法从噪声中提取有意义的预测信息。这不是因为"外部冲击太多"，而是因为**信号的频率与噪声的频率重叠**——WMA 交叉试图捕捉的"趋势"在 BTC 上并不存在，或者存在的时间尺度远长于交叉信号的响应时间。



## 5. 方法论教训


| 问题 | 错误做法 | 正确做法 |
|------|---------|---------|
| 评估协议 | 单次切分 | 前向验证 |
| 优化目标 | 单窗口最优 | 多窗口平均（鲁棒优化） |
| 算法选择 | GP（树结构过拟合严重） | PSO（连续参数更稳健） |
| 基准 | 买入并持有（beta） | 信息比率 / Jensen Alpha（alpha） |
| 窗口大小 | 过小（<1年） | ≥3年训练（覆盖完整制度周期） |



## 6. 下一步


如果希望在这个框架内取得进展，合理的方向是：
1. **在线学习 / 自适应参数**：每窗口重新训练（walk-forward 已做，但需更短的训练期 + 更频繁的再训练）
2. **制度检测优先**：用马尔可夫转换模型或隐马尔可夫模型先检测当前制度，然后以制度为条件选择策略参数 [1]
3. **更高阶特征**：放弃 WMA 交叉，转向波动率、订单流、链上数据等更高维度的信号 [2]


### GP 正则化（Parsimony Pressure）

GP 的过拟合可以通过**简洁压力（Parsimony Pressure）**来缓解：

```python
fitness = cash - λ * tree_size
```

- `tree_size` = 树节点数
- `λ` = 惩罚系数（通常取现金范围的 1–5%）
- 效果：迫使 GP 偏好更简单的树，减少"记忆噪声"的能力

其他正则化手段：
- **树深度硬限制**：当前 max_depth=5，可以降到 3
- **Niching（生态位）**：维护种群多样性，防止所有树收敛到同一个过拟合解
- **Early stopping**：在验证集上监控性能，不再提升时停止

但需要注意：正则化只能缓解过拟合，不能改变**搜索空间本身的表达能力**。如果底层信号（WMA 交叉）无法捕捉 BTC 的动态，再简洁的树也只是"简洁地错误"。
