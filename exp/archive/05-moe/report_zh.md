# 实验 05：Mixture-of-Experts (MoE)

## 实验目的

如果不同市场制度需要不同参数，Mixture-of-Experts 是否能通过制度检测 + 专家分配来超越单一策略？

## 实验设计

1. **制度分割**：K-means 聚类将数据分为高波动/低波动两个 regime
2. **专家训练**：每个 regime 独立训练一个 trivial SMA 专家
3. **路由器**：波动率阈值决定使用哪个专家
4. **PSO 优化**：优化路由器阈值

## 结果

| 策略 | 测试收益 |
|------|----------|
| MoE | $478 |
| 无 MoE | $606 |

**MoE 不仅无益，反而有害**（$478 < $606）。

## 关键发现

### 1. MoE 失败的三个原因

**原因一：K-means 打乱时间顺序**
K-means 基于波动率聚类，不考虑时间顺序。高波动 regime 可能包含 2017 年和 2022 年的数据，这两个时期的市场行为完全不同。

**原因二：专家数据太少**
每个专家仅训练于 ~730 天数据，远少于完整训练集的 2190 天。数据不足导致过拟合。

**原因三：制度切换时的虚假交易**
当 regime 切换时，两个专家可能给出相反信号。backtester 在切换时产生未在训练中优化的虚假交易。

### 2. MoE 的理论动机是正确的

市场阶段分析（Exp 新实验）证明：不同市场阶段确实需要不同策略。MoE 的失败在于**实现方式**，而非**概念本身**。

## 讨论

### 未来改进方向

1. **使用隐马尔可夫模型（HMM）进行 regime detection**：考虑时间序列的转移概率
2. **为每个 regime 分配专门的策略族**：如趋势跟踪 vs 均值回归
3. **让路由器基于波动率和趋势强度 jointly 决策**

## 结论

1. **本实验的 MoE 实现失败**：$478 < $606
2. **失败原因**：K-means 打乱时间顺序、专家数据太少、切换时虚假交易
3. **MoE 的理论动机是正确的**：不同 regime 确实需要不同策略
4. **未来的改进方向**：HMM regime detection + 专门策略族
