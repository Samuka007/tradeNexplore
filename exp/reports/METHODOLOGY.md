# 方法论

## 1. 数据

- **来源**: Yahoo Finance BTC-USD
- **时间范围**: 2014-01-01 至 2022-12-31
- **频率**: 日数据
- **训练集**: <2020-01-01（约 2190 天）
- **测试集**: 2020-01-01 至 2022-12-31（约 1095 天）

## 2. 回测引擎

### 离散信号

信号 ∈ {-1, 0, 1}，触发全仓买入/卖出：
- 1: 买入（全部现金 → BTC）
- -1: 卖出（全部 BTC → 现金）
- 0: 持仓不变

### 连续仓位控制

信号 ∈ [0, 1]，表示持仓比例：
- 0: 空仓
- 1: 全仓
- (0,1): 部分持仓

仓位调整按 delta 比例进行，每次调整支付手续费。

### 手续费

固定 3% 双边手续费（买入 3%，卖出 3%）。

###  fitness 函数

默认: `final_cash`（最终现金）

## 3. PSO（粒子群优化）

### 算法参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| n_particles | 30 | 粒子数 |
| max_iter | 50 | 最大迭代数 |
| w | 0.9→0.4 | 惯性权重线性衰减 |
| c1, c2 | 2.05 | 认知/社会系数 |

### 变体测试

- **惯性策略**: linear decay, fixed 0.7, Clerc constriction
- **拓扑结构**: global best, ring local
- **粒子/迭代 trade-off**: 固定 1500 次评估，改变比例

## 5. 评估协议

### 5.1 单次切分

- 训练: <2020
- 测试: 2020-2022
- **问题**: 容易过拟合到特定市场阶段

### 5.2 Walk-forward

- 滚动 3 年训练 / 1 年测试 / 1 年步进
- **修正**: 原代码步进 train_years，已修正为 test_years
- **结果**: 5 个窗口，更真实的泛化估计

### 5.3 鲁棒优化

- 52 个窗口同时优化
- 适应度 = 随机 12 窗口平均收益
- **问题**: GP 退化为"不交易"策略

### 5.4 多种子验证

- 10 个随机种子（0, 7, 13, 21, 42, 55, 77, 88, 99, 123）
- 验证结论的稳健性

## 6. 策略设计

### 6.1 dual_crossover

- 参数: (d_fast, d_slow)
- 信号: {-1, 0, 1}
- 逻辑: sma_fast > sma_slow → 做多，反之做空

### 6.2 trivial_sma

- 参数: (d_fast, d_slow)
- 信号: {-1, 0, 1}
- 与 dual_crossover 类似，但实现不同

### 6.3 position_sma

- 参数: (d_fast, d_slow, scale)
- 信号: [0, 1] 连续仓位
- 逻辑: sigmoid(sma_fast - sma_slow, scale)

### 6.4 macd

- 参数: 7 个（fast, slow, signal, 等）
- 信号: {-1, 0, 1}

### 6.5 GP tree

- 参数: 树结构本身
- 信号: 树求值结果 > 0 → 做多，< 0 → 做空
- 连续模式: sigmoid(树求值结果)

## 7. 统计指标

| 指标 | 公式 | 用途 |
|------|------|------|
| 最终收益 | cash_end | 主要 fitness |
| 夏普比率 | mean(returns)/std(returns) × √252 | 风险调整收益 |
| 最大回撤 | min((equity - peak)/peak) | 风险度量 |
| 胜率 | wins/trades × 100% | 交易质量 |
| 交易次数 | len(trades) | 手续费影响 |
